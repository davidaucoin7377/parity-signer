51/// Errors directly related to transaction parsing
52///
53/// Signable transactions are differentiated based on prelude:
54///
55/// - `53xx00` mortal transactions
56/// - `53xx02` immortal transactions
57/// - `53xx03` text message transactions
58///
59/// `53xx00` and `53xx02` transactions contain encoded transaction data, and
60/// are parsed prior to signing using the network metadata. Transaction is
61/// generated in client, for certain address and within certain network.
62/// To parse the transaction and to generate the signature, Vault must
63/// have the network information (network specs and correct network metadata)
64/// and the public address-associated information in its database.
65///
66/// `53xx00` and `53xx02` transcations consist of:
67///
68/// - prelude, `53xx00` or `53xx02`, where `xx` stands for the encryption
69///   algorithm associated with address and network used
70/// - public key corresponding to the address that can sign the transaction
71/// - encoded call data, the body of the transaction
72/// - extensions, as set in the network metadata
73/// - genesis hash of the network in which the transaction was generated
74///
75/// Parsing process first separates the prelude, public key, genesis hash and
76/// the combined call + extensions data.
77///
78/// The call information is SCALE-encoded into `Vec<u8>` bytes and then those
79/// bytes are SCALE-encoded again, so that the call data contained in the
80/// transaction consists of `compact` with encoded call length in bytes
81/// followed by the `Vec<u8>` with the encoded data.
82///
83/// Call and extensions are cut based on the call length declared at the start
84/// of the combined call + extensions data.
85///
86/// Then the extensions are decoded, and it is checked that the metadata version
87/// in extensions coincides with the metadata version used for the decoding.
88///
89/// Decoding the extensions for metadata with `RuntimeMetadataV12` or
90/// `RuntimeMetadataV13` is using a static set of extensions, namely:
91///
92/// - [`Era`](https://docs.rs/sp-runtime/6.0.0/sp_runtime/generic/enum.Era.html)
93/// - nonce, compact `u64`
94/// - transaction tip, compact `u128`
95/// - metadata version, `u32`
96/// - tx version, `u32`
97/// - network genesis hash, `[u8; 32]`
98/// - block hash, `[u8; 32]`
99///
100/// Decoding the extensions for metadata with `RuntimeMetadataV14` uses
101/// dynamically acquired set of extensions from the metadata itself.
102///
103/// After the extensions, the call data itself is decoded using the network
104/// metadata. Each call first byte is the index of the pallet.
105///
106/// Metadata with `RuntimeMetadataV12` or `RuntimeMetadataV13` has only type
107/// names associated with call arguments. Vault finds what the types really
108/// are and how to decode them by using the types information that must be in
109/// Vault database.
110/// For `RuntimeMetadataV12` or `RuntimeMetadataV13` the second byte in call is
111/// the index of the method within the pallet, and thes Vault finds the types
112/// used by the method and proceeds to decode the call data piece by piece.
113///
114/// Metadata with `RuntimeMetadataV14` has types data in-built in the metadata
115/// itself, and the types needed to decode the call are resolved during the
116/// decoding. For `RuntimeMetadataV14` the second byte in call is also
117/// the index of the method within the pallet, but this already goes into the
118/// type resolver.
119///
120/// Calls may contain nested calls, for `RuntimeMetadataV12` or
121/// `RuntimeMetadataV13` metadata the call decoding always starts with pallet
122/// and method combination processing. For `RuntimeMetadataV14` metadata the
123/// nested calls are processed through the type resolver, i.e. the pallet index
124/// is processed independently only on the start of the decoding.
125///
126/// `53xx03` transaction consists of:
127///
128/// - prelude `53xx03`, where `xx` stands for the encryption algorithm
129///   associated with address and network used
130/// - public key corresponding to the address that can sign the transaction
131/// - SCALE-encoded `String` contents of the message
132/// - genesis hash of the network in which the transaction was generated
133///
134/// Vault assumes that every byte of the transaction will be processed, and
135/// shows an error if this is not the case.
136#[derive(Debug)]
137pub enum ParserDecodingError {
138    /// Transaction was announced by the prelude to be mortal (`53xx00`),
139    /// but has `Era::Immortal` in extensions
140    UnexpectedImmortality,
141
142    /// Transaction was announced by the prelude to be immortal (`53xx02`),
143    /// but has `Era::Mortal(_, _)` in extensions
144    UnexpectedMortality,
145
146    /// Genesis hash cut from the end of the transaction does not match the one
147    /// found in the extensions
148    GenesisHashMismatch,
149
150    /// In immortal transaction the block hash from the extensions is the
151    /// network genesis hash.
152    ///
153    /// This error happens when block hash is different with the genesis hash
154    /// cut from the end of the transaction.
155    ImmortalHashMismatch,
156
157    /// Error decoding the extensions using metadata with `RuntimeMetadataV12`
158    /// or `RuntimeMetadataV13`, with default extensions set.
159    ExtensionsOlder,
160
161    /// Used only for `RuntimeMetadataV12` or `RuntimeMetadataV13`,
162    /// indicates that method index (second byte of the call data) is not valid
163    /// for the pallet with found name.
164    MethodNotFound {
165        /// index of the method, second byte of the call data
166        method_index: u8,
167
168        /// name of the pallet, found from the first byte of the call data
169        pallet_name: String,
170    },
171
172    /// Used only for all calls in `RuntimeMetadataV12` or `RuntimeMetadataV13`,
173    /// and for entry call in `RuntimeMetadataV14` metadata. First byte of the
174    /// call data is not a valid pallet index.
175    ///
176    /// Associated data is what was thought to be a pallet index.
177    PalletNotFound(u8),
178
179    /// Only for entry call in `RuntimeMetadataV14`. Pallet found via first byte
180    /// of the call has no associated calls.
181    ///
182    /// Associated data is the pallet name.
183    NoCallsInPallet(String),
184
185    /// Only for `RuntimeMetadataV14`. Found type index could not be resolved
186    /// in types registry
187    V14TypeNotResolved,
188
189    /// Only for `RuntimeMetadataV12` and `RuntimeMetadataV13`. Argument type
190    /// could not be taken out of `DecodeDifferent` construction.
191    ArgumentTypeError,
192
193    /// Only for `RuntimeMetadataV12` and `RuntimeMetadataV13`. Argument name
194    /// could not be taken out of `DecodeDifferent` construction.
195    ArgumentNameError,
196
197    /// Parser was trying to find an encoded
198    /// [`compact`](https://docs.rs/parity-scale-codec/latest/parity_scale_codec/struct.Compact.html),
199    /// in the bytes sequence, but was unable to.
200    NoCompact,
201
202    /// Parser was expecting more data.
203    DataTooShort,
204
205    /// Parser was unable to decode the data piece into a primitive type.
206    ///
207    /// Associated data is primitive identifier.
208    PrimitiveFailure(String),
209
210    /// SCALE-encoded `Option<_>` can have as a first byte:
211    ///
212    /// - `0` if the value is `None`
213    /// - `1` if the value is `Some`
214    /// - `2` if the value is `Some(false)` for `Option<bool>` encoding
215    ///
216    /// This error appears if the parser encounters something unexpected in the
217    /// first byte of encoded `Option<_>` instead.
218    UnexpectedOptionVariant,
219
220    /// Only for `RuntimeMetadataV12` and `RuntimeMetadataV13`.
221    /// Decoding
222    /// [`IdentityFields`](https://docs.substrate.io/rustdocs/latest/pallet_identity/struct.IdentityFields.html)
223    /// requires having correct type information for
224    /// [`IdentityField`](https://docs.substrate.io/rustdocs/latest/pallet_identity/enum.IdentityField.html)
225    /// in types information. If types information has no entry for
226    /// `IdentityFields` or it is not an enum, this error appears.
227    IdFields,
228
229    /// Parser processes certain types as balance (i.e. transforms the data
230    /// into appropriate float using decimals and units provided).
231    /// For some types the balance representation is not possible, this error
232    /// occurs if the parser tried to process as a balance some type not
233    /// suitable for it.
234    BalanceNotDescribed,
235
236    /// SCALE-encoded enum can have as a first byte only correct index of the
237    /// variant used.
238    ///
239    /// This error appears if the first byte is an invalid variant index.
240    UnexpectedEnumVariant,
241
242    /// Parser found that type declared as a
243    /// [`compact`](https://docs.rs/parity-scale-codec/latest/parity_scale_codec/struct.Compact.html)
244    /// has inner type that could not be encoded as a `compact`
245    UnexpectedCompactInsides,
246
247    /// Only for `RuntimeMetadataV12` and `RuntimeMetadataV13`.
248    /// Parser has encountered a type that could not be interpreted using the
249    /// existing types information.
250    ///
251    /// Associated data is the type description as it was received by parser
252    /// from the metadata.
253    UnknownType(String),
254
255    /// Only for `RuntimeMetadataV14`.
256    /// While decoding
257    /// [`BitVec<T,O>`](https://docs.rs/bitvec/1.0.0/bitvec/vec/struct.BitVec.html),
258    /// parser encountered `T` type not implementing
259    /// [`BitStore`](https://docs.rs/bitvec/1.0.0/bitvec/store/trait.BitStore.html).
260    NotBitStoreType,
261
262    /// Only for `RuntimeMetadataV14`.
263    /// While decoding
264    /// [`BitVec<T,O>`](https://docs.rs/bitvec/1.0.0/bitvec/vec/struct.BitVec.html),
265    /// parser encountered `O` type not implementing
266    /// [`BitOrder`](https://docs.rs/bitvec/1.0.0/bitvec/order/trait.BitOrder.html).
267    NotBitOrderType,
268
269    /// Only for `RuntimeMetadataV14`.
270    /// Parser failed to decode
271    /// [`BitVec<T,O>`](https://docs.rs/bitvec/1.0.0/bitvec/vec/struct.BitVec.html),
272    /// even though `T` and `O` types were suitable.
273    BitVecFailure,
274
275    /// Only for `RuntimeMetadataV14`.
276    /// Parser failed to decode data slice as
277    /// [`Era`](https://docs.rs/sp-runtime/6.0.0/sp_runtime/generic/enum.Era.html).
278    Era,
279
280    /// Parser expects to use all data in decoding. This error appears if some
281    /// data was not used in parsing of the method.
282    SomeDataNotUsedMethod,
283
284    /// Only for `RuntimeMetadataV14`.
285    /// Parser expects to use all data in decoding. This error appears if some
286    /// data from extensions is not used in the decoding.
287    SomeDataNotUsedExtensions,

290/// Errors occurring because the network metadata
291/// [`RuntimeMetadataV14`](https://docs.rs/frame-metadata/15.0.0/frame_metadata/v14/struct.RuntimeMetadataV14.html)
292/// has extensions not compatible with Vault.
293///
294/// To be compatible with signable transactions, metadata extensions must
295/// include:
296///
297/// - `Era` (once)
298/// - block hash (once)
299/// - metadata version (once)
300/// - network genesis hash (at most, once)
301///
302/// If the metadata does not follow those criteria, transactions could not be
303/// parsed, and therefore, could not be signed.
304#[derive(Debug)]
305pub enum ParserMetadataError {
306    /// Metadata extensions have no `Era`
307    NoEra,
308
309    /// Metadata extensions have no block hash
310    NoBlockHash,
311
312    /// Metadata extensions have no network metadata version
313    NoVersionExt,
314
315    /// Metadata extensions have more than one `Era`
316    EraTwice,
317
318    /// Metadata extensions have more than one genesis hash
319    GenesisHashTwice,
320
321    /// Metadata extensions have more than one block hash
322    BlockHashTwice,
323
324    /// Metadata extensions have more than one network metadata version
325    SpecVersionTwice,
